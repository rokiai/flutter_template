# Flutter MVVM Riverpod — 目录结构（精简模板）

## 总览

本仓库是 **纯模板**：计数 Demo、主题/多语言、Dio 接口示例。按 Feature 垂直拆分，Feature 内 MVVM。

```
lib/
├── main.dart
├── constants/
├── environment/          # Envied + Flavor
├── extensions/
├── features/
│   ├── common/           # Dio、主题/语言 Provider、通用组件
│   ├── counter/          # 计数 demo
│   ├── api_example/      # GET /posts 示例
│   ├── main/             # 底部 Tab
│   ├── onboarding/       # Splash
│   └── profile/          # 外观 + 语言设置
├── l10n/
├── routing/
├── theme/
└── utils/
```

## Feature 约定

```
feature_name/
├── model/
├── repository/
└── ui/
    ├── xxx_screen.dart
    ├── state/
    ├── view_model/
    └── widgets/
```

## 路由

| Path | 页面 |
|------|------|
| `/` | Splash → Main |
| `/main` | Counter / API / Settings |
| `/appearances` | 主题 |
| `/languages` | 语言 |

## 接口示例说明

- `PostsRepository` 使用 `dioProvider`，`baseUrl` 来自 `Env.apiBaseUrl`
- 默认指向 JSONPlaceholder，无需 Token
- UI：`ApiExampleScreen` → `PostsViewModel` → `PostsRepository`
